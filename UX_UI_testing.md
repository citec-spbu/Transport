# Отчет по UX и UI тестированию

## Объект исследования

Объектом исследования является web-приложение **Transit Network**,
предназначенное для анализа городских транспортных сетей. Система
собирает данные о транспортных потоках, сохраняет их в виде графов в
базе данных Neo4j, выполняет кластеризацию и вычисление различных метрик
центральности. Приложение ориентировано на интерактивное взаимодействие
с пользователем и поддерживает пошаговый сценарий выбора параметров
анализа с последующим отображением результатов в виде кластеризаций и
тепловых карт.

Пользовательский интерфейс приложения реализован в виде
последовательности экранов, каждый из которых соответствует отдельному
этапу пользовательского сценария. 

**Основной сценарий работы с приложением
включает следующие шаги:**

***Начальный экран***

При входе в приложение пользователю отображается стартовая страница,
содержащая название приложения и кнопку начала работы. Данный экран
выполняет навигационную функцию и служит точкой входа в основной
пользовательский сценарий. К нему можно вернуться с любой страницы
приложения если нажать на название приложения в левом верхнем углу.

***Экран выбора города и вида транспорта***

После начала работы пользователь переходит к экрану выбора параметров
анализа. На данном этапе интерфейс включает:

-   поле ввода названия города с механизмом автодополнения, предлагающим доступные варианты;

-   группу элементов выбора вида транспорта (транспорт автобус, трамвай и др.), реализованную в виде переключателей (радиокнопок), допускающих выбор только одного варианта;

-   кнопку перехода к следующему этапу, которую можно нажать (она активна) лишь при вводе доступного города и выбора вида транспорта.

При нажатии на кнопку перехода интерфейс изменяет своё состояние: текст
кнопки заменяется на сообщение о загрузке данных, а также отображается
анимация загрузки. В этот момент выполняется получение данных из кеша
или их загрузка с внешнего источника.

***Экран выбора метрик***

После завершения загрузки данных пользователь переходит к экрану выбора
типа метрик для анализа. Интерфейс позволяет выбрать один из двух
режимов:

-   кластеризационные метрики;

-   тепловые карты (метрики центральности).

Выбор осуществляется с помощью переключателей, аналогичных используемым
при выборе вида транспорта. Переход к следующему этапу сопровождается
изменением состояния кнопки (отображение процесса обработки данных) и
анимацией загрузки.

***Экран визуализации кластеризаций***

В случае выбора кластеризационного анализа пользователь переходит на
экран визуализации результатов. Интерфейс данного экрана включает:

-   отображение выбранного города;

-   переключатель типа кластеризации (алгоритмы Лувена и Лейдена);

-   числовые показатели, отражающие количество узлов (остановок) и число кластеров;

-   кнопку отображения легенды, содержащей соответствие номеров кластеров их цветам и количество узлов в группах в скобках;

-   интерактивную карту, реализованную с использованием библиотеки Leaflet.

На карте узлы транспортной сети визуализируются с цветовой
дифференциацией по принадлежности к кластерам. При выборе узла
отображается дополнительная информация, включающая название узла, номер
кластера и используемый алгоритм кластеризации.

***Экран визуализации тепловых карт***

При выборе режима тепловых карт пользователю отображается альтернативный
экран визуализации, включающий:

-   выбор метрики центральности (PageRank или Betweenness);

-   информацию о выбранном городе, количестве узлов и максимальном значении метрики;

-   элемент экспорта данных, позволяющий сохранить результаты в форматах JSON или CSV. В них для каждого узла указывается ID, имя, номер группы для кластеризации или значение метрики для тепловой карты, координаты.

Основную часть экрана занимает интерактивная карта Leaflet, на которой
узлы окрашены в соответствии с градиентом значений выбранной метрики.
При выборе узла отображается информация о его названии, значении метрики
и координатах.

Дополнительно интерфейс включает:

-   шкалу цветового градиента, отображающую переход от минимальных к максимальным значениям метрики;

-   столбчатую диаграмму, отображающую значения метрики для узлов, отсортированных по убыванию;

-   элементы управления диаграммой, позволяющие задавать количество отображаемых столбцов и начальную позицию в отсортированном списке;

-   кнопку сброса параметров диаграммы к значениям по умолчанию;

-   кнопку разворота диаграммы в полноэкранный режим и выхода из него;

-   кнопку для скрытия диаграммы и информации о городе, максимальном значении метрики и количестве узлов;

-   кнопки стрелки для изменения начальной позиции в отсортированном списке на единицу за нажатие;

-   при наведении курсора компьютерной мыши на столбец появляется панель с именем узла, который ему соответствует, цвет, тип метрики, значение.

Таким образом, объект исследования представляет собой многошаговое
web-приложение с развитым пользовательским интерфейсом, включающим
навигационные элементы, формы ввода, интерактивные карты и средства
визуализации данных. Наличие различных режимов работы, состояний
интерфейса и элементов управления делает данное приложение подходящим
для комплексного тестирования функциональных возможностей, навигации,
содержимого страниц и удобства использования пользовательского
интерфейса.

# UI тестирование

UI тестирование web-приложения **Transit Network** было выполнено с
целью подтверждения корректности отображения и качества визуальных и
текстовых элементов пользовательского интерфейса с точки зрения
конечного пользователя. Как пример корректной ситуации использовалось
описание из главы «Объект исследования». Проверка проводилась для всех
основных экранов приложения, соответствующих этапам пользовательского
сценария.

### 1. Проверка отображения изображений и графических элементов

В рамках UI тестирования была выполнена проверка корректности
отображения изображений и графических элементов пользовательского
интерфейса web-приложения **Transit Network**. Проверка проводилась
путём последовательного прохождения всех этапов пользовательского
сценария в различных браузерах: **Mozilla Firefox**, **Google Chrome** и
**Konqueror**.

В ходе тестирования оценивалось отображение интерактивных карт, маркеров
узлов транспортной сети, цветовой дифференциации кластеров, тепловых
градиентов значений метрик центральности, а также диаграмм и
вспомогательных графических элементов интерфейса.

По результатам проверки установлено, что во всех используемых браузерах
графические элементы отображаются корректно. Визуальных искажений,
отсутствующих элементов, ошибок масштабирования или различий в структуре
отображения выявлено не было.

### 2. Соответствие содержимого страниц требованиям

В рамках UI тестирования была выполнена проверка соответствия
отображаемых элементов пользовательского интерфейса пользовательскому
сценарию работы с приложением. Проверка проводилась путём
последовательного прохождения всех этапов сценария и сопоставления
фактически отображаемых элементов интерфейса с перечнем элементов,
заявленных в описании приложения.

На **начальном экране** проверялось наличие названия приложения и кнопки
начала работы, обеспечивающей переход к основному пользовательскому
сценарию.

На **экране выбора города и вида транспорта** проверялось наличие поля
ввода названия города с механизмом автодополнения, элементов выбора вида
транспорта, а также кнопки перехода к следующему этапу.

На **экране выбора метрик анализа** проверялось наличие элементов выбора
режима анализа (кластеризационные метрики и тепловые карты) и элементов
управления запуском обработки данных.

На **экране визуализации кластеризаций** проверялось наличие
интерактивной карты, переключателей алгоритмов кластеризации, числовых
показателей, элементов отображения легенды и информационных всплывающих
окон.

На **экране визуализации тепловых карт** проверялось наличие элементов
выбора метрики центральности, интерактивной карты, шкалы цветового
градиента, столбчатой диаграммы, элементов управления диаграммой и
средств экспорта данных.

По результатам проверки установлено, что все элементы интерфейса,
предусмотренные пользовательским сценарием работы приложения,
присутствуют и отображаются корректно. Отсутствие обязательных элементов

ходе тестирования не выявлено.

В рамках UI тестирования была проведена проверка стабильности структуры
пользовательского интерфейса при различных сценариях взаимодействия.

встроенных в браузер кнопок навигации «вперёд» и «назад».

В ходе тестирования оценивалось сохранение расположения основных
элементов интерфейса, включая заголовки, кнопки управления,
интерактивные карты, диаграммы и информационные панели. Также
проверялось отсутствие смещения, перекрытия или исчезновения элементов
при смене экранов и возврате к предыдущим этапам пользовательского
сценария.

По результатам проверки установлено, что структура страниц остаётся
стабильной во всех проверенных браузерах. Переходы между экранами,
включая использование стандартных средств навигации браузера, не
приводят к нарушению компоновки интерфейса, потере элементов или
искажению отображаемого содержимого.

По результатам проверки установлено, что при переключении между
алгоритмами кластеризации и метриками анализа ранее загруженные данные и

некорректной работе приложения, однако может снижать наглядность
текущего контекста для пользователя.

Также было отмечено, что информация о выбранном виде транспорта
отображается только на этапе его выбора и не представлена на последующих

системы, отсутствие явного указания выбранного вида транспорта на
экранах анализа может затруднять восприятие результатов, особенно при
повторном использовании приложения или сравнении различных сценариев
анализа.

Отдельно была проверена корректность текстового содержимого, связанного
с экспортом результатов анализа. Установлено, что при экспорте данных в
форматы JSON и CSV в названии файлов и структуре экспортируемых данных
не указывается информация о выбранной метрике анализа. Данная
особенность не нарушает корректность самих данных, однако может снижать
удобство их последующего использования и идентификации.

В целом текстовое содержимое интерфейса является точным и
непротиворечивым, а выявленные особенности отображения информации не
влияют на работоспособность приложения. Вместе с тем указанные моменты
могут рассматриваться как потенциальные направления для улучшения
пользовательского опыта за счёт более явного отображения выбранных
параметров анализа.

### 10. Проверка орфографии, грамматики и терминологии

В рамках UI тестирования была проведена проверка орфографии, грамматики

использованием кратких и лаконичных текстовых формулировок, состоящих,

При анализе используемой терминологии было отмечено, что некоторые

В целом используемая в интерфейсе терминология соответствует предметной
области анализа транспортных сетей и не содержит ошибок, однако
отдельные обозначения параметров визуализации могут быть уточнены для
повышения понятности пользовательского интерфейса.

UX тестирование проводилось методом сценарного анализа с наблюдением за
